# C 언어 배열에서 `*` 와 `+1` 연산의 의미 정리
## 1차원 배열 vs 2차원 배열

---

## 1. 핵심 개념 요약

- 배열의 이름은 **배열 전체의 시작 주소**이다.
- 포인터 연산(`+1`)의 의미는 **가리키는 타입에 따라 달라진다**.
- `*` 연산자는 “값을 하나 꺼낸다”가 아니라  
  **“포인터가 가리키는 대상 전체를 참조한다”** 는 의미이다.

---

## 2. 1차원 배열에서의 해석

### 2.1 선언

```c
char f[5] = "abcd";
````

### 2.2 각 표현의 의미

| 표현         | 타입       | 의미                    |
| ---------- | -------- | --------------------- |
| `f`        | `char *` | 첫 번째 원소의 주소 (`&f[0]`) |
| `f + 1`    | `char *` | 두 번째 원소의 주소 (`&f[1]`) |
| `*f`       | `char`   | 첫 번째 원소의 값 (`f[0]`)   |
| `*f + 1`   | `int`    | 첫 번째 원소의 값 + 1        |
| `*(f + 1)` | `char`   | 두 번째 원소의 값 (`f[1]`)   |

### 2.3 핵심 관계식

```c
f + 1      == &f[1]
*(f + 1)   == f[1]
*f         == f[0]
*f + 1     == f[0] + 1   // 값 연산
```

📌 **1차원 배열에서 `*f`는 곧 `f[0]`의 값이다.**

---

## 3. 2차원 배열에서의 해석

### 3.1 선언

```c
char f[3][5];
```

### 3.2 타입의 차이 (중요)

| 표현         | 타입            |
| ---------- | ------------- |
| `f`        | `char (*)[5]` |
| `*f`       | `char[5]`     |
| `f[0]`     | `char[5]`     |
| `f[0] + 1` | `char *`      |

---

## 4. 2차원 배열에서 `*` 와 `+1`

### 4.1 `f + 1`

```c
f + 1
```

* 한 행(`char[5]`) 단위 이동
* → **2행의 시작 주소**
* 값: `&f[1]`

---

### 4.2 `*f`

```c
*f
```

* 의미: **0행 전체**
* 타입: `char[5]`
* 식에서 사용되면 `char *`로 decay
* 값: `&f[0][0]`

---

### 4.3 `*f + 1`

```c
*f + 1
```

* 의미: `&f[0][0] + 1`
* → **0행 1열의 주소**

```c
*f + 1 == f[0] + 1 == &f[0][1]
```

📌 **2차원 배열에서는 `*f + 1`이 값 연산이 아니라 주소 연산이다.**

---

### 4.4 값이 필요할 때

```c
*(*f + 1) == f[0][1]
```

또는

```c
**f == f[0][0]
```

---

## 5. 1차원 vs 2차원 비교

| 표현       | 1차원 배열   | 2차원 배열      |
| -------- | -------- | ----------- |
| `f`      | 첫 원소 주소  | 전체 배열 시작 주소 |
| `f + 1`  | `&f[1]`  | 다음 행 주소     |
| `*f`     | `f[0]` 값 | 0행 전체       |
| `*f + 1` | 값 + 1    | 0행 1열 주소    |
| 값 접근     | `*f`     | `**f`       |

---

## 6. 핵심 공식 정리 (암기용)

```c
// 1차원 배열
f + i      == &f[i]
*(f + i)   == f[i]
*f + 1     == f[0] + 1   // 값 연산

// 2차원 배열
f + i          == &f[i]
*(f + i)       == f[i]
*(*f + j)      == f[0][j]
*f + j         == &f[0][j]
**f            == f[0][0]
```

---

## 7. 최종 한 줄 요약

> `*`와 `+1`의 의미는 연산자 자체가 아니라
> **그 순간의 타입이 무엇이냐에 따라 완전히 달라진다.**
