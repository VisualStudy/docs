# C 언어 배열 표기와 포인터 해석 정리
## 1차원 배열부터 2차원 배열까지

---

## 1. 배열의 기본 개념

- C 언어에서 배열은 **연속된 메모리 공간**이다.
- 배열의 이름은 **배열 전체의 시작 주소**를 의미한다.
- 배열의 모든 인덱스는 **0부터 시작**한다.

---

## 2. 1차원 배열

### 2.1 선언

```c
char a[5];
````

### 2.2 의미

| 표현         | 의미          |
| ---------- | ----------- |
| `a`        | 배열 시작 주소    |
| `a[0]`     | 첫 번째 원소의 값  |
| `&a[0]`    | 첫 번째 원소의 주소 |
| `a + 1`    | 두 번째 원소의 주소 |
| `*(a + i)` | `a[i]`와 동일  |

### 2.3 핵심 관계식

```c
a[i] == *(a + i)
```

---

## 3. 2차원 배열

### 3.1 선언

```c
char f[3][5];
```

* 총 3행, 각 행은 char 5개로 구성됨
* **배열의 배열** 구조

---

### 3.2 메모리 구조 개념

```
f
├─ f[0] → f[0][0] f[0][1] f[0][2] f[0][3] f[0][4]
├─ f[1] → f[1][0] f[1][1] f[1][2] f[1][3] f[1][4]
└─ f[2] → f[2][0] ...
```

---

## 4. 2차원 배열에서의 타입과 의미

### 4.1 각 표현의 타입과 역할

| 표현        | 타입                | 의미          |
| --------- | ----------------- | ----------- |
| `f`       | `char (*)[5]`     | 전체 배열 시작 주소 |
| `f + 1`   | `char (*)[5]`     | 2행 시작 주소    |
| `f[i]`    | `char[5] → char*` | i행 시작 주소    |
| `f[i][j]` | `char`            | i행 j열 값     |

---

## 5. 포인터 연산 관점에서의 해석

### 5.1 행 단위 이동

```c
f + 1
```

* 한 행(`char[5]`) 단위로 이동
* → 2행 시작 주소

---

### 5.2 행 접근

```c
*(f + i) == f[i]
```

* i번째 행 전체를 의미
* 식에서는 `char*`로 사용됨

---

### 5.3 열 접근

```c
f[i][j] == *(f[i] + j) == *(*(f + i) + j)
```

---

## 6. 0행 접근을 f만으로 표현하기

### 6.1 0행 전체

```c
*f == f[0]
```

* 0행 전체(`char[5]`)
* 식에서는 `char*`로 decay

---

### 6.2 0행 j열 값

```c
(*f)[j]
*(*f + j)
```

모두 다음과 동일:

```c
f[0][j]
```

---

## 7. 1차원 vs 2차원 배열 비교

| 구분      | 1차원 배열   | 2차원 배열      |
| ------- | -------- | ----------- |
| 배열명     | 원소 시작 주소 | 전체 배열 시작 주소 |
| `+1` 의미 | 다음 원소    | 다음 행        |
| `*` 의미  | 원소 값     | 한 행         |
| 인덱스     | `a[i]`   | `f[i][j]`   |

---

## 8. 행과 열의 인덱스 기준

* 모든 배열 인덱스는 **0부터 시작**
* 2차원 배열에서도 동일

```c
f[0][0]  // 0행 0열
f[1][2]  // 1행 2열
f[2][4]  // 마지막 원소
```

---

## 9. 핵심 요약 (시험용)

* 배열명은 배열 전체의 시작 주소이다.
* 2차원 배열에서 배열명은 **행 배열을 가리키는 포인터**이다.
* `f[i]`는 i번째 행의 시작 주소이다.
* `f[i][j]`는 `*(*(f + i) + j)`와 동일하다.
* 모든 행과 열의 인덱스는 0부터 시작한다.
